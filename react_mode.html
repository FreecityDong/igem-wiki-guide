一、项目总览

本项目基于 React + TypeScript + Vite 构建，是一个模块化的前端网站，适合 iGEM 等文档类展示用途。

二、核心技术栈

  • React：用于构建组件化页面结构
  • TypeScript：增强类型安全与开发体验
  • Vite：快速构建工具，支持热重载
  • Yarn：依赖管理工具（也支持 npm）
  • GitLab CI/CD：自动部署与测试

三、目录结构说明

根目录配置文件

  • .env：定义运行时环境变量
  • .eslintrc.cjs：配置 ESLint 静态代码分析规则
  • .gitignore：指定 Git 应忽略的文件和目录
  • .gitlab-ci.yml：GitLab 持续集成与部署配置
  • README.md：项目使用与说明文档
  • LICENSE：开源许可证声明（如 MIT）
  • index.html：HTML 模板入口，React App 挂载点
  • package.json：项目元数据和依赖声明
  • vite.config.ts：Vite 构建和插件配置
  • tsconfig.json：TypeScript 主配置文件
  • tsconfig.node.json：Node.js 环境的补充配置
  • yarn.lock：锁定依赖版本，确保一致性

src/ 目录结构

  • main.tsx：项目入口，React 应用挂载至 DOM
  • App.tsx：主应用容器，组织页面结构和路由
  • App.css：App 的样式文件
  • pages.ts：页面路径与组件的映射，用于路由控制
  • vite-env.d.ts：Vite 自动生成的类型声明文件

src/components/：基础组件

  • Footer.tsx：底部页脚组件
  • Header.tsx：顶部标题组件
  • Navbar.tsx：导航栏组件
  • Inspirations.tsx：灵感参考内容展示
  • NotFound.tsx：404 页面组件

src/contents/：每个页面内容组件

  • home.tsx、description.tsx、model.tsx 等 iGEM 页面

src/utils/：工具函数

  • getPathMapping.ts：获取路由与组件映射关系
  • stringToSlug.ts：将字符串转 URL 友好的 slug
  • index.ts：工具函数总出口

四、本地部署步骤

# 安装依赖
yarn install

# 启动开发服务器
yarn dev

# 构建生产版本
yarn build

五、适用场景

  • 科研展示（iGEM、课题组主页）
  • 团队介绍与作品集
  • 轻量级静态文档网站

可根据实际需求灵活扩展页面结构与组件内容。
